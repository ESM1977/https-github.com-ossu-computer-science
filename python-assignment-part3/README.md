# File I/O, APIs & Exception Handling

This project is based on understanding how Python works with files, APIs, and error handling.  
It is designed to perform basic file operations, fetch data from an API, and handle different types of errors without crashing. Four different tasks are performed:

## Task 1 — File Read & Write Basics
Part A: This part creates a file named python_notes.txt and writes some python notes into it
Part B: This part reads the file back and prints each line with numbering, it counts total number of lines and searches for a user-provided keyword

## Task 2 - API Integration
This task uses the requests library with the DummyJSON Products API.

## Task 3 — Exception Handling
This task demonstrates safe error handling in Python.

Part A — Guarded Calculator
Part B — Guarded File Reader
Part C — Robust API Calls
Part D — Input Validation Loop

## Task 4 — Logging to File
This task builds a simple error logger.

## Files in this folder

- part3_api_files.py → main program  
- python_notes.txt → generated after running the code  
- error_log.txt → contains logged errors  
- README.md → this file  

## Note

The text files are created only after running the program.  
So make sure the code is executed completely before uploading.
